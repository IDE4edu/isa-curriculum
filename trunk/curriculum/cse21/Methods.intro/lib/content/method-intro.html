What are Methods?

Methods are Java’s mechanism for breaking down a computation into smaller (and, we hope, more manageable) pieces. (Methods are also known as subprograms.) This kind of decomposition is analogous to splitting a book into chapters or sorting documents into folders in a file cabinet.

Every method has

  • a header which contains three pieces of information:
    • a name by which the method is accessed,
    • a type that indicated why type of data the method will pass back, and
    • a list of parameters that describe the information to be given to the method when it runs (parameters allow methods to do different things in different circumstances)
  • a body of statements that perform the computation associated with the method.

When a program is run and encounters a call to or an invocation of a method—using its name—the program executes the statements in the method body and then resumes execution at the statement immediately following the call.

We’ll start with methods that have return types of void, meaning they don't return anything, and have empty parameter lists, meaning the method will always run the same way.
