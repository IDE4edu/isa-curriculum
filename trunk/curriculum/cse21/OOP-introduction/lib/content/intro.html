The basics: Classes and Objects in Java

In Object-oriented programs, a programmer uses combinations of separate objects to perform the work of the program. An object is a self-contained 'thing' that can have properties and behaviors. A class is a template, or blueprint, for objects of a particular kind (i.e., that belong to that class).

An object can have instance variable—regular Java variables that belong to the object—and represent properties of the object. Different objects of the same kind (i.e., that were made using the same class) will have different values for these instance variables.

An object can also have methods that represent behaviors the object can perform.

In this assignment, we'll work with a Card class that represents typical playing card as used in games like poker.
