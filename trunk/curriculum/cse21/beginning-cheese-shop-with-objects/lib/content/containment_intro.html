The Containment design: when one object controls other objects

A very common pattern of object-oriented programs involves containment. In these programs, an object of one type contains an object (or objects) of another type (or types). This is common enough in the real world:

  • A refrigerator could hold Milk, produce objects of different types, and so forth, in its different compartments.
  • A deck of cards contains cards in a certain order.
  • An address book contains addresses, perhaps grouped into categories.

Containment implies more than the simple linking of objects: it implies that the container has some control over the objects that it contains, invoking their behaviors in a privileged way, creating and deleting the objects, keeping track of what it contains, and so forth.

In this assignment, you will program a Cheese class that stores certain information about itself, and a Store class that contains certain Cheese objects.
